# Contribution Guide

このリポは個人運営の設計テンプレート集です。貢献は歓迎しますが、以下のポリシーを先に確認してください。

---

## 言語ポリシー

- **日本語のみで対応**します
- 英語の issue / PR は基本的に対応しません（無視されることがあります）
- ドキュメント・コメントも日本語で記述

理由: 元素材が日本の現場業務由来で、ニュアンスが英語に乗り切らないため。
英語コミュニティ向けには [The Agency](https://github.com/msitarzewski/agency-agents) 等の既存リポを推奨。

---

## 応答速度

- 個人運営のため、返信が遅れます（数日〜1週間）
- 即座の対応は期待しないでください

---

## どんな貢献を歓迎するか

### ◎ 歓迎

- ロールカードのカスタマイズ事例（業界別・規模別）の共有
- 引用強要ルールの拡張パターン
- Codex Review Gate の実装例
- ドキュメントのタイポ修正・誤訳修正
- 自組織で運用してみた失敗事例（落とし穴の追加）

### ○ 検討

- 新しいロール（CMO/CIO 等）の追加
- 多言語化（日本語以外への翻訳・ただし上記言語ポリシー優先）

### × 受けない

- このリポを「人格カタログ」化する変更（[The Agency](https://github.com/msitarzewski/agency-agents) と被るため別リポで）
- 元素材（特定案件）の内部詳細推測・公開要請
- 「もっと自動化しろ」「もっとエージェント増やせ」系の方向性転換

## issue / PR のフォーマット

### issue

```markdown
## 状況
（再現できる状況）

## 期待
（期待した動作）

## 実際
（実際の動作）

## 引用元
（このリポのどのファイル・どの行を参照したか）
```

### PR

```markdown
## 変更内容
（一行サマリ）

## なぜこの変更が必要か
（背景・問題意識）

## 検証方法
（変更後の確認手順）

## 機密チェック
- [ ] 個人名・組織名が含まれていないか
- [ ] 業界特定の表現が紛れていないか
- [ ] 既存のロールカード構造を壊していないか
```

---

## 機密情報のチェック

PR には必ず `.claude/hooks/pre_commit_scan.sh` を通してください:

```bash
bash .claude/hooks/pre_commit_scan.sh --all
```

検出された場合は、汎化または削除してから再コミット。

---

## 行動規範

- **過剰な称賛・過剰な否定はしない**（振り子の法則）
- **「いかがでしたか」「画期的な」「ぜひ」等の AI 臭い定型文を避ける**
- **引用なき主張は推測扱いとして扱う**（このリポの中心思想）
- **不可逆操作の前は明示的選択を待つ**（破壊的提案を出したら、ユーザーの選択を待つ）
