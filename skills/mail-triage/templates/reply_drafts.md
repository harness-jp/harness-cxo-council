# Reply Drafts — PB別返信ドラフト型

> 8 PB それぞれの返信骨格テンプレート。
> 「人間が削れば OK」前提の素朴版。完璧を目指さない。

---

## 設計の起源

LLM にゼロから返信を書かせると、毎回トーンがブレる。
**PB ごとに骨格を固定**し、変数だけ埋めて出力する方が速くて安定する。

「人間が削れば OK ベース」: ドラフトは多めに書いておき、人間が送信前に**削る前提**。
LLM が追加するより、人間が削る方が安全（情報漏洩・誤約束を防ぐ）。

---

## 共通スタイルルール

全 PB 共通で守る:

| ルール | 内容 |
|---|---|
| 短く素直 | 1メール = 100〜300字目安 |
| 装飾語ゼロ | 「誠に」「謹んで」「差し支えなければ」「恐縮ながら」を使わない |
| 感謝+依頼 同時 NG | 「ありがとうございます」と即「ご連絡ください」をつなげない（[skip_rules.md L3](../prompts/skip_rules.md) 違反） |
| 自分の署名 | 末尾に固定署名（{SIGNATURE} で置換） |
| 件名 | `Re:` を保持、件名本体は変えない |

---

## 置換ポイント

各テンプレで以下を `{}` で囲んで明示:

| 置換変数 | 意味 |
|---|---|
| `{SENDER_NAME}` | 送信者の呼び方（さん付け / 様付け） |
| `{SUBJECT_REF}` | 元の件名・話題への言及 |
| `{KEY_INFO}` | 確定情報（日時・金額・条件等） |
| `{NEXT_ACTION}` | 次のアクション（送ります / お待ちします） |
| `{INDUSTRY_TERM}` | 業界固有語（書類名・サービス名等） |
| `{SIGNATURE}` | 自分の署名 |

## P-01: 新規問合せ・初動対応

### 骨格

```
{SENDER_NAME}

お問合せありがとうございます。
{SUBJECT_REF}の件、確認しまして折り返しご連絡いたします。

{NEXT_ACTION_TIMING}までにお返事いたします。

{SIGNATURE}
```

### 変数例

- `{NEXT_ACTION_TIMING}`: 「本日中」「明日午前中」「3営業日以内」

### 注意

- 「ご連絡ください」を入れない（向こうがすでに連絡してきた）
- 即答できない場合の「いつまでに」を必ず明示

---

## P-03: 担当者変更・引継ぎ

### 骨格

```
{SENDER_NAME}

ご連絡ありがとうございます。
{ORIGINAL_OWNER_NAME}より引き継ぎ、本件の担当となりました{SELF_NAME}と申します。

{TAKEOVER_STATUS}まで把握しております。
{NEXT_ACTION}

引き続きよろしくお願いいたします。

{SIGNATURE}
```

### 変数例

- `{TAKEOVER_STATUS}`: 「直近の{INDUSTRY_TERM}」「先月の合意内容」

### 注意

- 引き継ぎ漏れがある場合は素直に「{XX}について再度ご教示いただけますか」と書く
- 「不明な点はお問合せください」のような曖昧な依頼を避ける

---

## P-04: 現場確認・引渡し

### 骨格

```
{SENDER_NAME}

{SUBJECT_REF}の件、承知しました。

日時: {DATE_TIME}
場所: {LOCATION}
持ち物: {ITEMS}

当日よろしくお願いいたします。
{NEXT_ACTION}

{SIGNATURE}
```

### 変数例

- `{ITEMS}`: 「印鑑、身分証、{INDUSTRY_TERM}」「特になし」

### 注意

- 立会系は**事実情報の確認**が主。装飾語ほぼ不要
- 当日変更可能性があれば「変更時はXXまでご連絡ください」を1行追加

---

## P-05: 日程調整

### 骨格

```
{SENDER_NAME}

{SUBJECT_REF}の件、ありがとうございます。

候補日時:
- {DATE_OPTION_1}
- {DATE_OPTION_2}
- {DATE_OPTION_3}

ご都合のよろしい日時を{NEXT_ACTION_TIMING}までにご教示ください。

{SIGNATURE}
```

### 変数例

- `{NEXT_ACTION_TIMING}`: 「明日中」「今週中」

### 注意

- 候補は3つ提示が目安（少ないと再調整・多いと選びづらい）
- 「ご教示ください」より「お知らせください」の方が軽い（自組織のトーンで選択）

## P-08: 重要書類送付（⚠️ D-021 no_autosend 強制）

### 骨格

```
{SENDER_NAME}

{DOCUMENT_TYPE}をお送りいたします。

ご確認のうえ、{REVIEW_DEADLINE}までにご返信いただけますと幸いです。

{KEY_POINTS}

{SIGNATURE}
```

### 変数例

- `{DOCUMENT_TYPE}`: 「契約書ドラフト」「覚書案」「見積書」
- `{KEY_POINTS}`: 重要条項の要約（金額情報は本文に書かない、別ファイルで）

### 注意（最重要）

- **金額・違約金・賃料等の具体額は本文に書かない**（誤送信時の被害規模が桁違い）
- **必ず Codex Gate (`mail_send` 操作) を通す**
- 添付の差し替え忘れチェック必須

---

## P-09: 電子サイン・記録

### 骨格

```
{SENDER_NAME}

{DOCUMENT_NAME}の{SIGN_OR_RECORD}の件、承知しました。

{DOCUMENT_LINK}

{NEXT_ACTION}

{SIGNATURE}
```

### 変数例

- `{SIGN_OR_RECORD}`: 「電子サイン」「議事録共有」「押印」
- `{NEXT_ACTION}`: 「本日中に対応いたします」「{DATE}までに完了予定です」

### 注意（D-021 強制）

- **証跡が残る業務 = 後戻り不可**。`no_autosend` 強制
- リンクの URL は 必ず正規 URL（短縮URLや偽装ドメインに注意）

---

## P-13: 契約終了に伴う作業

### 骨格

```
{SENDER_NAME}

{SUBJECT_REF}の件、承知しました。

{TASK_LIST}
完了予定: {COMPLETION_DATE}

ご不明点ございましたら{NEXT_ACTION_TIMING}までにご一報ください。

{SIGNATURE}
```

### 変数例

- `{TASK_LIST}`: 箇条書きで「鍵返却 / {INDUSTRY_TERM} / 立会」等

### 注意

- 「ご一報ください」は使う（[skip_rules L3](../prompts/skip_rules.md)で警告対象）
  - ⚠️ 「ありがとうございます」と組み合わせて使うと違反
  - 単独で使うのはOK

---

## P-14: 条件変更通知（⚠️ D-021 no_autosend 強制）

### 骨格

```
{SENDER_NAME}

{CONDITION_TYPE}の変更について、ご連絡いたします。

変更内容:
- 変更前: {BEFORE_VALUE}
- 変更後: {AFTER_VALUE}
- 変更日: {EFFECTIVE_DATE}

ご確認のうえ、{REVIEW_DEADLINE}までにご返信ください。

{SIGNATURE}
```

### 変数例

- `{CONDITION_TYPE}`: 「単価」「面積」「契約期間」「サービス内容」

### 注意（最重要）

- **数値を本文に書く必要がある場合、二重チェック必須**（OCR誤認識等で誤数値が紛れる）
- **必ず Codex Gate を通す**
- 変更履歴を `memory/decisions_ledger.md` に D-XXX として記録

## カスタマイズポイント

| 箇所 | 何を書き換えるか |
|---|---|
| 各 PB の骨格 | 自組織のトーンで定型部分を調整 |
| 共通スタイルルール | 業界文化（より丁寧 / よりカジュアル）に合わせる |
| 置換変数 | 自組織で頻出する変数を追加 |
| 装飾語の NG リスト | 自分が嫌う語を `prompts/skip_rules.md` と整合させて追加 |
