# P-04: 現場確認・引渡し — 例2（検収依頼）

## 期待される処理を先に

- **PB**: P-04（現場確認・引渡し）
- **信頼度**: 0.88

## 受信メール

```
件名: 納品物の検収のお願い
From: ito@example.com
To: contact@example.org
日時: 2026-05-26 13:50
```

```
ご担当者様

ABC商事の伊藤です。

5/24に納品させていただいた○○の検収をお願いいたします。
検収期限は契約上、6/7までとなっております。

検収完了後、検収書（添付の様式）にご記入の上、ご返送ください。
不備や不明点ございましたら、お早めにお知らせください。

何卒よろしくお願いいたします。

ABC商事 伊藤五郎
```

## 期待される処理

- **判定根拠**:
  - 件名「検収」+ 本文「納品物」「検収期限」「検収書」
  - 物理的な引渡し後のアクション = P-04
- **skip判定**:
  - L1 越境誤爆: 通過
  - L2 沈黙: 通過
  - L3 上から目線: draft 後に評価
- **no_autosend**: false（受領 + 進捗返信）
- **PRECEDENCE**: 単独該当
- **重要**: 検収期限（6/7）あり → 未対応のまま放置すると契約上の問題

## 期待される返信ドラフト

```
ABC商事 伊藤様

ご連絡ありがとうございます。

納品物の検収、承知しました。
社内で確認の上、6/7までに検収書をお戻しいたします。

不明点が出ましたら、別途ご連絡いたします。

{SIGNATURE}
```

## カスタマイズメモ

- 検収期限は**必ず復唱**して相手に伝える（自分のスケジュール感も同時に確定）
- 「不備や不明点ございましたら」の依頼は受けるが、自分の返信では「不明点が出たら」に書き換え（責任の所在を明確に）
- 「お早めに」「お早急に」は使わない（相手に圧をかけずに自分のペースを守る）
- 状況によっては社内タスク化必須 → `status.md#todo` 等に記録
