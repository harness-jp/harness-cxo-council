# P-08: 重要書類送付 — 例2（見積書送付）⚠️ no_autosend

## 受信メール

```
件名: お見積書のご送付
From: sales@example.com
To: contact@example.org
日時: 2026-05-29 11:30
添付: 御見積書_2026-05-29.pdf
```

```
ご担当者様

ABC商事 営業部の井上と申します。
お世話になっております。

先日ご依頼いただきました件のお見積書を、添付にてお送りいたします。

主な記載項目:
- 提案範囲: ○○サービス
- 提案期間: 2026-07-01 〜 2027-06-30（12ヶ月）
- 金額: 添付PDF記載のとおり
- 有効期限: 2026-06-30まで

ご検討のほど、よろしくお願い申し上げます。

ご質問や追加要件などございましたら、お気軽にご連絡ください。

ABC商事 営業部
井上九郎
```

## 期待される処理

- **PB**: P-08（重要書類送付）
- **信頼度**: 0.95
- **判定根拠**:
  - 件名「見積書」+ 添付に見積書PDF
  - 金額・有効期限あり = 重要書類
- **skip判定**:
  - L1 越境誤爆: 通過
  - L2 沈黙: 通過
  - L3 上から目線: draft 後に評価
- **no_autosend**: ⚠️ **true**（D-021 強制・金額情報含む）
- **PRECEDENCE**: P-08 最優先（金額情報が含まれるため）
- **追加処理**:
  - OCR で金額抽出 → 別 skill で社内承認フローへ
  - 有効期限（2026-06-30）を `memory/decisions_ledger.md` に記録
  - 比較見積が必要なら別ベンダーへの問合せもタスク化

## 期待される返信ドラフト（人間最終承認必須）

```
ABC商事 井上様

ご連絡ありがとうございます。

お見積書、受領いたしました。
内容を社内で確認の上、改めてご連絡いたします。

ご質問が出ましたら、別途お問合せいたします。

{SIGNATURE}
```

## カスタマイズメモ

- ⚠️ **絶対に自動送信しない**
- **金額の具体額を返信本文に書かない**（漏洩リスクと、相手から見ても「何回も金額をやり取りする」のは不自然）
- 「ご検討のほど、よろしくお願いします」のような相手の依頼に対しては「確認の上、改めて」と返す（即決を匂わせない）
- 比較見積を取る場合の社内タスク化が必要 → status.md / decisions_ledger 等で管理
