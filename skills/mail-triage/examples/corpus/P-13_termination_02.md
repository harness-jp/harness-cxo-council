# P-13: 契約終了に伴う作業 — 例2（引き継ぎ清掃）

## 受信メール

```
件名: 6/30 退去・引継ぎ清掃のご相談
From: building@example.com
To: contact@example.org
日時: 2026-06-04 11:00
```

```

ABC商事 物件管理部の渡辺です。
お世話になっております。

6/30 退去予定の物件について、引継ぎ清掃の手配をご相談いたします。

- 物件: 東京都○○区 ××ビル 3階
- 退去日: 2026-06-30
- 清掃希望日: 2026-06-28 〜 2026-06-29
- 立会者: 弊社渡辺、貴社1名

清掃業者の手配は弊社にて進めますが、貴社側で特別な希望（業者指定・時間帯指定等）ございましたら

ABC商事 物件管理部

```

## 期待される処理

- **PB**: P-13（契約終了に伴う作業）
- **信頼度**: 0.91
- **判定根拠**:
  - 件名「退去」「引継ぎ清掃」+ 本文「退去日」「立会者」
  - 物理的な引き渡しを伴う終了フロー
- **skip判定**:
  - L1 越境誤爆: 通過
  - L2 沈黙: 通過
  - L3 上から目線: draft 後に評価
- **no_autosend**: false（清掃手配の応答）
- **PRECEDENCE**: P-13 確定（P-04 立会要素を含むが、終了フローが主目的）
  - 注: P-04（現場確認）と紛らわしいが、「退去」「終了」の文脈なので P-13 を選ぶ

## 期待される返信ドラフト

```
ABC商事 渡辺様

ご連絡ありがとうございます。

退去・引継ぎ清掃の件、承知しました。

清掃業者の指定や時間帯希望は特にございません。
6/15 までに変更があればご連絡いたします。

立会日時の最終確定をお待ちしております。

{SIGNATURE}
```

## カスタマイズメモ

- 「特になし」と明確に答える（曖昧にすると相手が再質問する）
- 6/15 までの「変更あれば連絡」は自分の社内タスク化必須（カレンダーに6/14リマインダー設定）
- 立会日時の最終確定を「お待ちしております」で締める = 相手のボール明示
- 物件・退去・清掃のような物理アクション系は P-04 に間違いやすい → 「終了」「退去」が主軸なら P-13 を選ぶ（PRECEDENCE 参照）
