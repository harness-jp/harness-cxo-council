# P-03: 担当者変更・引継ぎ — 例2（退職挨拶）

## 受信メール

```
件名: 退職のご挨拶
From: nakamura@example.com
To: undisclosed-recipients
日時: 2026-05-23 17:30
```

```
お取引先各位

平素より大変お世話になっております。
ABC商事の中村と申します。

私事で恐縮ですが、5/31をもちまして、ABC商事を退職することとなりました。
これまでのご厚意に心より感謝申し上げます。

6/1以降の本件の担当は、同部署の渡辺（watanabe@example.com）が引き継ぎます。
既に申し送り済みですので、引き続きよろしくお願いいたします。

略儀ながらメールにてご挨拶申し上げます。

ABC商事 中村三郎
```

## 期待される処理

- **PB**: P-03（担当者変更・引継ぎ）
- **信頼度**: 0.94
- **判定根拠**:
  - 件名「退職のご挨拶」+ 本文「退職」「引き継ぎ」
  - undisclosed-recipients (BCC一括送信) → 個別案件ではなく挨拶系
- **skip判定**:
  - L1 越境誤爆: 通過
  - L2 沈黙: 通過
  - L3 上から目線: draft 後に評価
- **no_autosend**: false（短い御礼返信）
- **PRECEDENCE**: 単独該当

## 期待される返信ドラフト

```
ABC商事 中村様

ご連絡ありがとうございます。
そして長らくお世話になりました。

新天地でのご活躍をお祈りしております。
渡辺様への引き継ぎ、承知いたしました。

{SIGNATURE}
```

## カスタマイズメモ

- 退職挨拶は**短く返す**。長文は逆に重い
- 「長らくお世話になりました」が定番
- BCC一斉送信メールには返信不要、と判断する組織もある（自組織の文化次第）
- 場合によっては skip 判定にする運用もアリ（個人で返信する余裕がない場合）
